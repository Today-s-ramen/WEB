## Today's Ramen

### Branch 전략

1. 이슈 생성
2. feat/#issue-number\_기능
3. 작업 완료 후 main 브랜치로 PR 생성

### 커밋 컨벤션

```
[init]: init
[feat]: 기능 추가
[update]: 기능 보완
[refactor]: 리팩토링
[fix]: 버그 수정
[design]: 스타일 코드 작성
[chore]: 애매한데 중요하지는 않은 작업
[asset]: 이미지, 폰트 등 리소스 추가
[docs]: 문서 (Readme 수정)
```

### 역할 분배
